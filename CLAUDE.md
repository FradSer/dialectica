# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

Dialectica is a reasoning-engine toolbox built on Google ADK. The shipped public API is now just two things, both proven by controlled evals: the **`Workflow` kernel** (`workflow.py` — `agent`/`parallel`/`pipeline`/`phase`/`log`/`budget`), whose `agent(tools=...)` primitive lets a stage do what a single forward pass cannot (act → observe → iterate; measured 8/8 vs a single call's 0/8 for a small model on tasks requiring interaction), and **execution-guided repair** (`repair.py`, verifier-in-the-loop: best-of-N reliability at ~1/3 the cost on verifiable tasks, now rebuilt on top of the kernel). Everything else this project built — a dedicated agentic-engine class, a heterogeneous ensemble + float scorer, a dialectic (thesis → antithesis → synthesis) engine, and a legacy Tree-of-Thoughts + GAN baseline — was measured to either need nothing beyond `agent(tools=...)` or to tie/lose a prompt-matched single call as a pure-LLM scaffold, so they were demoted to unshipped reference patterns in `examples/patterns/`. Hard-won finding: no pure-LLM scaffold beats a prompt-matched single call on *self-contained* tasks — engines win by adding capability (tools), ground-truth verification, or (on open-ended meta-tasks) heterogeneous model independence. The measured open-ended recipe is `examples/patterns/reflection_pattern.py` (`create_reflection_engine`) — still not a third shipped engine. Published on PyPI as `dialectica`. Python 3.11+.

## Commands

- Install deps: `uv sync`
- Test (mocked, fast, no API key): `uv run pytest`
- Test live E2E (calls real Gemini, needs `GOOGLE_API_KEY`): `uv run pytest -m e2e` — auto-skipped without the key
- Eval (engine vs single-call baseline, real API): `uv run python -m evals [--limit N --json out.json]`
- Open-ended / multi-model ablations (cliproxy): `uv run python -m evals.reflection_ablation`, `evals.workflow_ablation`, `evals.quality_workflow_ablation`
- Format / lint: `uv run ruff format` / `uv run ruff check`
- Never hand-edit `pyproject.toml`; use `uv add` / `uv remove`.

## Testing workflow

BDD-driven TDD: new behavior starts with a Gherkin scenario in `tests/features/*.feature`, executable via pytest-bdd — step definitions live in `tests/test_*_feature.py` (bound with `scenarios(...)`). Then RED test → GREEN code → REFACTOR. When updating tests, update the matching `.feature` first. This rigor is reserved for the shipped kernel + repair; `examples/patterns/` gets a lighter regression net (`tests/test_example_patterns_smoke.py`, one mocked end-to-end run per pattern) since it's reference code, not shipped API.

Mock the LLM at the single seam `agent_runtime.run_agent()` — never patch ADK internals or per-stage agents (`tests/helpers.py` has the fakes). `asyncio_mode = auto`, so plain async tests need no decorator; pytest-bdd steps are sync — wrap coroutines with `asyncio.run()`. Tests load `dialectica/.env` via `tests/conftest.py`.

CI (`.github/workflows/test.yml`) runs `ruff format --check`, `ruff check`, and `pytest` on every push/PR; the release workflow runs the same gates.

## Architecture

**Two shipped things, honest hierarchy** (controlled evals in `evals/results/`, README "Evaluation"):

- `workflow.py` — `Workflow` + `agent`/`parallel`/`pipeline`/`workflow`/`phase`/`log`/`budget`/`run_id`: programmatic port of Claude Code's Workflow tool (IDE host UI excluded). `agent(..., isolation="worktree", agent_type="Explore")`; `workflow(script_or_name)` child-workflow join (one nesting level); resume/journal via `Workflow(..., resume_run_id=...)` persisting to `.dialectica/workflows/<run_id>/`; `register_workflow` for named scripts; lifetime cap 1000 `agent()` calls, 4096 items per `parallel`/`pipeline`; optional `meta` phase validation. **`tools=[...]`** is the capability-add lever (8/8 vs 0/8 hidden-oracle). `budget` meters calls and tokens; `Workflow(budget_unit="tokens")` gates on output tokens. Pure schema-only workflows remain bound by negative quality findings.
- `repair.py` — `create_repair_engine`: verifier-in-the-loop, rebuilt on `workflow.py` via `wf.workflow(script)`. Child-workflow join charges outer `budget_total`. Reproduce with `evals/repair_ablation.py`.
- `workflow_journal.py`, `workflow_registry.py`, `workflow_worktree.py` — journal/resume, named workflow registry, git worktree isolation for `agent(isolation="worktree")`.

`agent_factory.py` builds ADK `LlmAgent`s from `ROLE_TEMPLATES` — trimmed to `{"Generator": ...}` only, since every shipped call site now uses `wf.agent(instructions=...)` instead of a Discriminator/Synthesizer role template. `agent_runtime.run_agent()` is the only place that actually calls the LLM; it retries transient failures with exponential backoff (`_call_agent_once` is the raw transport — retry tests patch that, everything else patches `run_agent`). ADK 2.3 wiring lives in `adk_config.py` + `_make_runner()`: set `DIALECTICA_CONTEXT_CACHE=true` to pass `App(context_cache_config=...)` into `InMemoryRunner` (helps multi-turn tool loops within one `agent()` call on Gemini); `TokenUsage.cached_tokens` surfaces `cached_content_token_count` from usage metadata; `DIALECTICA_ADK_TELEMETRY=true` or standard `OTEL_EXPORTER_OTLP_*` env vars trigger `maybe_set_otel_providers()` on the first real call. `json_repair.py` holds the two shared fence/escape-repair helpers `workflow.py` depends on (all that survived from the former `gan_evaluator.py` once its GAN-specific classes were demoted). The library never calls `logging.basicConfig` — the consuming app owns logging setup.

**`examples/patterns/`** (a dev tool, not shipped in the wheel, same as `evals/`) holds runnable reference implementations — demoted engines plus the measured open-ended recipe — rebuilt on `workflow.py` primitives (plain functions/closures over `agent()`/`parallel()`), not a Protocol-based plugin system:

- `reflection_pattern.py` — **canonical open-ended recipe** `create_reflection_engine`: hetero gather → frame → critique → synthesize with per-angle `model=`. Measured **5-0-0** vs single/homo on meta (finding #6); **10-0-0** vs single on meta+default (finding #7). Lever is roster heterogeneity, not a scorer. Prefer this over other quality modes.
- `quality_workflow_pattern.py` — mode switcher (`reflection` / `adversarial` / `dialectic`) for ablation; `mode="reflection"` delegates to `reflection_pattern`. Finding #7: adversarial/dialectic add no consistent lift over hetero reflection — keep for comparison only.
- `agentic_pattern.py` — `create_agentic_engine`/`AgenticEngine`: wraps `agent(task, tools=tools, instructions=AGENTIC_SYSTEM_PROMPT...)` in a `Workflow` script. Same 8/8 vs 0/8 as the kernel primitive; copy-pasteable recipe, not a needed class.
- `dialectic_pattern.py` — `create_dialectic_engine`/`DialecticEngine`: thesis → antithesis → synthesis spiral, scoring via `agent(schema=Verdict)`. Ties/loses a prompt-matched single call (**0-3-2**); content-steering + auditable trace only.
- `ensemble_pattern.py` — `create_ensemble_engine`/`EnsembleSearchEngine`: AB-MCTS-lite over a heterogeneous roster, mandatory float `scorer`. **CUT** — blind-pick matched the scorer's robustness gain; signal adds nothing over heterogeneity alone.
- `tot_gan_pattern.py` — `create_engine`/`create_coordinator`/`Coordinator`: beam search + GAN-style refinement; sibling expand/evaluate via `wf.parallel`. **Measured dominated**; baseline only.

Top-level `evals/` is the eval harness (dev tool, not shipped): each problem is solved by the engine and a single-call baseline, then a blind judge compares both answers twice with swapped positions — disagreement is a tie. `harness.py` counts LLM calls through the `run_agent` seam. Model overrides: `BASELINE_MODEL_CONFIG` / `JUDGE_MODEL_CONFIG` / `GENERATOR_MODEL_CONFIG`. Key ablations: `repair_ablation`, `agentic_eval`, `ensemble_ablation` / `ensemble_meta_ablation`, `reflection_ablation`, `workflow_ablation`, `quality_workflow_ablation` (five-arm mode comparison on meta+default). Cliproxy live defaults: `OPENAI_API_BASE`/`OPENAI_API_KEY` from `CLIPROXYAPI_*`, `GENERATOR_MODEL_CONFIG=openai:qwen3.6-flash`, `JUDGE_MODEL_CONFIG=openai:glm-5.2`, `DIALECTICA_DISABLE_THINKING=true`.

Eval findings (2026-06, three 3-run × 5-problem matrices, see README "Results"): with the original "Innovation" discriminator criterion (V1) the engine won technical problems 7-1-1 but lost organizational ones 0-4-2 ("over-complex"), reproducing across Gemini and Qwen. Replacing "Innovation" with "Feasibility under stated constraints" (V2 + second seed V3) pooled to 20-8-2 (technical 15-2-1, organizational 5-6-1) vs V1's 7-5-3. Key insight: because the GAN loop refines thoughts against the critique, discriminator criteria steer answer content, not just selection. Engine cost ≈ 20-30× baseline calls; concurrency cut wall-clock 2-3× at unchanged call counts. Reports land in `evals/results/` (gitignored).

Prompt-controlled finding (2026-06-15, qwen3.6-35b-a3b via CLIPROXYAPI, blind position-swap) — **why repair is the core**: strengthening the dialectic (synthesis "dominate a single expert pass" + thesis baseline-strength) beat a *naively*-prompted single call (4-1-0) and plain ToT (pooled 8-2-0), but against a **prompt-matched** strong baseline it went **0-3-2 (loses) at equal length** — the gain was the prompt, not the structure. Lesson: pure-LLM scaffolds (ToT, GAN, dialectic) only rearrange the model's own thinking on one context, so they tie a single call; a scaffold beats one pass only by adding information a single pass lacks — tool-grounding (`repair.py`), scale, or independence. On verifiable tasks the band where repair beats a single call is *fails-but-fixable*, and it shrinks as the model strengthens (this model solved HumanEval 17-18/18 and curated LCB-hard at pass@1, leaving little gap to demonstrate).

SWE-suite findings (2026-06, ground truth): against a 1-attempt baseline the engine looked +1 (11/12 vs 10/12 on gpt-oss:20b; 10/12 vs 9/12 on gemma4:e4b), but rescue mode with a pass@2 screen dissolved it — the apparent lift was resampling luck, and the single real gap (max-fill on e4b) was NOT rescued (full mode even regressed min-path on e4b). After extending to 18 problems with the literature's hardest HumanEval items, cloud gemma-4 (31b-it and 26b-a4b-it via AI Studio) solved 18/18 at pass@2 — HumanEval-class is saturated for this family; real code-value evidence needs LiveCodeBench/competition difficulty. Methodology rules: compare scaffolds against pass@k at matched cost; the advice suite still needs a best-of-2-plus-judge control. Backend quirk: enforced JSON mode breaks gemma-4-26b-a4b-it (empty/truncated verdicts) — the ToT+GAN pattern's `structured_output=False` toggle for this is documented but not actually wired to anything after the demotion (see pattern docstring).

2026-06-30 consolidation (this file's own honesty gate): Agentic (75 LOC) and Repair (236 LOC) were the only two ✅-verified wins; `AgenticEngine`'s unique logic was ~15-19 lines (a system prompt + a name), and `create_agent()`/`workflow.agent()` already supported `tools=` generically — so Agentic was folded entirely into the kernel primitive rather than kept as a separate class. Ensemble (already ⚠️ CUT) and ToT+GAN (already ❌ dominated) were demoted alongside it to `examples/patterns/`, since neither was pulling weight as bespoke shipped engines. `workflow.agent()` gained `instructions=` (to preserve Agentic's system-prompt capability) and a fix for a latent bug where `model=` was never resolved through `_parse_model_config` (found during the port, never previously exercised since no test or eval called `wf.agent(model=...)`).

2026-07 reflection / quality-workflow findings (README #6 / #7): hetero multi-angle reflection is the measured open-ended lever (**5-0-0** on meta; **10-0-0** vs single on meta+default). Homogeneous reflection still beats single (**4-0-1** / **4-0-6**) but loses headroom to hetero. Extra adversarial-rival or one-round dialectic stages add no consistent lift over hetero reflection (adversarial **2-0-8**, dialectic **0-1-9** vs reflection). Do not resurrect AB-MCTS float scorers for meta-tasks; default callers use `create_reflection_engine`.

## Gotchas

- The library does **not** load `.env` — consuming apps own that. Only the test suite loads it (`dialectica/.env`). `evals/` scripts also do **not** auto-load `.env` — export cliproxy/`OPENAI_*` in the shell (see `evals/score_workflow.sh`).
- Models: use `gemini-3.5-flash` (default) or `gemini-3.1-pro-preview` only — there is no stable `gemini-3.1-pro` (404 on generateContent). Configure via `provider:model_name` env vars (`DEFAULT_MODEL_CONFIG`, plus optional `GENERATOR_MODEL_CONFIG` — the only role-specific override the shipped kernel reads, since `ROLE_TEMPLATES` only has `Generator` now). Parsed in `llm_config.py`.
- Env vars: `GOOGLE_API_KEY` for AI Studio, or the Vertex trio (`GOOGLE_GENAI_USE_VERTEXAI=true`, `GOOGLE_CLOUD_PROJECT`, `GOOGLE_CLOUD_LOCATION`). ADK 2.3 perf: `DIALECTICA_CONTEXT_CACHE`, `DIALECTICA_CONTEXT_CACHE_*` tunables, `DIALECTICA_ADK_TELEMETRY` / `OTEL_EXPORTER_OTLP_*`. See `.env.example` (repo root; copy to `dialectica/.env` for the test suite).
- `examples/patterns/*.py` are dev-tool reference code, not shipped in the wheel (`pyproject.toml` only packages `["dialectica"]`) — import them the way `evals/*.py` does (`from examples.patterns.reflection_pattern import create_reflection_engine`, etc.), not from the `dialectica` package.
- `wf.parallel()` needs a **list of thunks**, not a bare generator comprehension (`len()` is called on the argument).
- Passing `get_model_config()`'s `LiteLlm` object into roster maps is fine for `wf.agent(model=...)`, but heterogeneity checks must normalize via a hashable key (see `reflection_pattern._model_key`).
